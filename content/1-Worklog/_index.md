**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with AWS and basic AWS services](1.1-week1/)

**Week 2:** [Learned the basics of EC2, successfully deployed and connected instances, and understood AWS networking services.](1.2-week2/)

**Week 3:** [Understood AWS Compute VM services and successfully deployed AWS Backup, File Storage Gateway, S3 bucket, and Autoscaling Group.](1.3-week3/)

**Week 4:** [Learned AWS storage services, practiced VM Import/Export and deployed File Storage Gateway, completed the worklog, and finalized the project idea.](1.4-week4/)

**Week 5:** [Understood AWS security services and completed labs on EC2 cost optimization and resource management.](1.5-week5/)

**Week 6:** [Completed IAM labs, gained basic knowledge of AWS KMS, and explored AWS database services.](1.6-week6/)

**Week 7:** [Completed module 6 labs (RDS and database migration) and made progress on researching and implementing the group project.](1.7-week7/)

**Week 8:** [In the process of reviewing for the midterm exam.](1.8-week8/)

**Week 9:** [Reviewed and completed the midterm exam.](1.9-week9/)

**Week 10:** [Continued progress in researching and implementing the group project.](1.10-week10/)

**Week 11:** [Continued progress in researching and implementing the group project.](1.11-week11/)

**Week 12:** [Continued progress in researching and implementing the group project.](1.12-week12/)
